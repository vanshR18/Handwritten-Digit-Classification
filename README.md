## 🧠 Overview

An end-to-end digit recognition pipeline built with PyTorch:

- **EDA** — class distribution, sample images, pixel intensity analysis, writing style variation
- **Preprocessing** — normalization, train/val/test split, augmentation (rotation + affine)
- **Baseline CNN** — 3-block CNN trained from scratch
- **ResNet-18** — adapted for grayscale 28×28 input with modified first conv and removed MaxPool
- **Evaluation** — confusion matrix, per-class F1, misclassification analysis, model comparison
- **FastAPI** — REST API with auto-inversion support (works with both MNIST-style and paper-style images)

---

## 📂 Dataset

**MNIST** — 70,000 grayscale handwritten digit images.  
Downloaded automatically via `torchvision.datasets.MNIST` — no manual download needed.

| Property | Value |
|---|---|
| Total Images | 70,000 |
| Train / Val / Test | 54,000 / 6,000 / 10,000 |
| Image Size | 28 × 28 (grayscale) |
| Classes | 10 (digits 0–9) |
| Balance | ✅ Well balanced (~6,000–7,000 per class) |
| Source | Loaded via `torchvision.datasets.MNIST` |

                │

          ├── POST /predict       (single image)

## 📓 Running the Notebooks

```bash
jupyter lab
```

### Notebook 1 — EDA & Preprocessing
```
notebooks/01_EDA_Preprocessing.ipynb
```
- MNIST downloads automatically to `./data/` on first run
- Run all cells — takes ~2 minutes
- Saves `outputs/dataset_meta.json` + all EDA plots

### Notebook 2 — Model Training & Evaluation
```
notebooks/02_Baseline_CNN_ResNet_Evaluation.ipynb
```
- Loads `outputs/dataset_meta.json` from Notebook 1
- Trains Baseline CNN (15 epochs) then ResNet-18 (20 epochs)
- Saves `models/resnet_inference.pth`

> 💡 GPU recommended. Expected training times:
> - Baseline CNN: ~3 min on GPU, ~15 min on CPU
> - ResNet-18: ~8 min on GPU, ~35 min on CPU

## 🔗 API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/` | Welcome + endpoint list |
| `GET` | `/health` | Model status, device, uptime |
| `POST` | `/predict` | Single image → digit + confidence |
| `POST` | `/predict/batch` | Batch prediction (up to 16 images) |

> **Auto-inversion:** The API automatically detects and inverts images with white backgrounds (paper-style drawings), so both MNIST-style (white digit on black) and paper-style (black digit on white) images work correctly.

## 📤 Example Requests

### Single Image — `curl`
```bash
# Using a file saved to disk
echo '{}' > /dev/null  # just to show the pattern
curl -X POST "http://localhost:8000/predict" \
  -H "accept: application/json" \
  -F "file=@digit_image.png"
```

### Single Image — Python
```python
import requests

with open("digit_image.png", "rb") as f:
    response = requests.post(
        "http://localhost:8000/predict",
        files={"file": ("digit_image.png", f, "image/png")}
    )

data = response.json()
print(f"Predicted Digit : {data['predicted_digit']}")
print(f"Confidence      : {data['confidence_pct']}")
print(f"Inference time  : {data['inference_ms']} ms")
```

### Batch Prediction — Python
```python
import requests

files = [
    ("files", ("digit1.png", open("digit1.png", "rb"), "image/png")),
    ("files", ("digit2.png", open("digit2.png", "rb"), "image/png")),
    ("files", ("digit3.png", open("digit3.png", "rb"), "image/png")),
]

response = requests.post("http://localhost:8000/predict/batch", files=files)
data = response.json()

print(f"Total images : {data['total_images']}")
print(f"Total time   : {data['total_time_ms']} ms\n")

for result in data["results"]:
    print(f"{result['filename']:<20} → Digit {result['predicted_digit']}  ({result['confidence_pct']})")
```
